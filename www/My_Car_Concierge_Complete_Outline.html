3. User Types & Roles

Member (Vehicle Owner)

  • Create and manage vehicle profiles in Digital Garage
  • Submit service requests (Maintenance Packages)
  • Receive anonymous bids from providers
  • Accept bids with escrow-protected payments
  • Rate and review completed services
  • Access maintenance reminders and service history

Provider (Service Professional)

  • Complete verification and vetting process
  • Purchase bid credits to bid on jobs
  • Submit competitive bids with timeline estimates
  • Manage team members and certifications
  • Access performance analytics and ratings
  • Receive payments via Stripe

Pending Provider

  • Awaiting admin approval after application
  • Limited access until verification complete

Admin

  • Approve/reject provider applications
  • Manage user accounts and roles
  • Handle disputes and refunds
  • Process founder payouts
  • Monitor platform analytics

Dual Roles

Users can have both Member and Provider roles simultaneously, managed via is_also_member and is_also_provider flags.

7. Payment System

Payment Flow

  1. Bid Accepted: Member accepts provider bid
  2. Authorization: Stripe PaymentIntent created with capture_method: 'manual'
  3. Escrow: Card authorized but NOT charged (funds held)
  4. Work Completed: Provider marks job complete
  5. Member Confirmation: Member confirms work satisfaction
  6. Capture: Payment captured and distributed
  7. Provider Payout: Provider receives 92.5% via Stripe Connect

Stripe Products Used

  • Stripe Connect: Provider payouts via Express accounts
  • Payment Intents API: Authorization/capture flow for escrow
  • Webhooks: Async payment status updates
  • Checkout Sessions: Bid pack purchases

Dispute Handling

  • PaymentIntent cancelled, funds returned to member
  • Admin review required for dispute resolution
  • Bid credit refunds available when applicable
