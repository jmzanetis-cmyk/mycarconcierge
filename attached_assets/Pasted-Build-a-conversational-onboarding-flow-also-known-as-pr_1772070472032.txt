Build a conversational onboarding flow (also known as progressive profiling) 
for a React app called My Car Concierge. This is a mobile-first automotive 
service marketplace that connects vehicle owners with vetted local service 
providers through a competitive bidding system.

Visit https://www.mycarconcierge.com and review the existing app — the 
registration forms, member dashboard, and provider dashboard. Use what you 
find there to determine exactly what information needs to be collected during 
onboarding for both members and providers. Do not guess or invent fields — 
only ask for what the platform actually needs to function.

Build TWO separate conversational onboarding flows based on what you find:

  1. Member Onboarding — for vehicle owners signing up to request services
  2. Provider Onboarding — for service professionals signing up to bid on jobs

---

ONBOARDING RULES:

- This is conversational onboarding / progressive profiling — 
  one question per screen, no traditional forms
- Write each question in plain, friendly, conversational language — 
  like a person talking, not a database field label
- Group related fields logically but still surface one input at a time
- Only collect what is truly necessary to get started — 
  anything that can be gathered contextually later should be skipped
- On the final step of provider onboarding, hand off to a background 
  verification step (Checkr integration — placeholder button for now)

---

DESIGN REQUIREMENTS:

- Full screen per question, vertically centered
- Progress bar or dots at top showing completion percentage
- Smooth slide-left animation between steps
- Large, full-width "Next" button at the bottom
- Input auto-focuses when each question appears
- Pressing Enter/Return advances to the next step
- Friendly sans-serif font (Inter or system-ui)
- Color scheme: white background, dark text, 
  accent color #2563EB (blue) for buttons and progress indicators
- Mobile first — optimized for 390px width (iPhone 14)

---

TECHNICAL REQUIREMENTS:

- Pure React with hooks (useState, useEffect)
- No external libraries except lucide-react for icons if needed
- All collected answers stored in a single state object per flow
- On final step, console.log the complete collected object 
  (Supabase insert will be wired in separately later)
- Export MemberOnboarding and ProviderOnboarding as named 
  components from a single file: Onboarding.jsx
- Create App.jsx with two buttons to toggle between 
  the two flows for testing purposes

---

Review the site first, then build. Start with MemberOnboarding, 
confirm it works, then build ProviderOnboarding using the same base pattern.
